# **Цель проекта**

Разработка пайплайна для обработки, анализа и визуализации геномных данных человека с применением практик инженерии данных. Проект демонстрирует полный цикл работы с биологическими данными – от сырых аннотаций до интерактивных дашбордов.

### **Ключевые задачи**

1. **Загрузка данных** – автоматическое получение информации о генах из научных баз
2. **Очистка данных** – проверка и исправление ошибок в генетической информации
3. **Структурирование** – организация данных для удобного поиска и анализа
4. **Визуализация** – создание графиков и диаграмм для изучения генов

# **Установка и запуск**

### **Предварительные требования**

- Python 3.9+
- Poetry 1.4+
- 2 ГБ свободного места на диске

### **Установка и запуск**

```bash
# Клонирование репозитория
git clone https://github.com/wyrtnhw09-cpu/DataEngineeringITMO2025.git
cd DataEngineeringITMO2025

# Установка зависимостей через Poetry
poetry install

# Активация виртуального окружения
poetry shell

# Загрузка и предобработка данных
python src/data_loader.py

# Запуск интерактивного анализа
jupyter notebook notebooks/EDA.ipynb
```

# **Архитектура проекта**

### **Схема пайплайна обработки данных**

Источники данных → Извлечение → Валидация → Трансформация → Хранение → Визуализация

### **Компоненты пайплайна**

<div align="center">

| Этап | Компонент | Назначение |
|------|-----------|------------|
| **Извлечение** | `extract.py` | Загрузка данных из CSV и биологических репозиториев |
| **Валидация** | `validate.py` | Проверка целостности, качества и полноты данных |
| **Трансформация** | `transform.py` | Очистка, преобразование и обогащение данных |
| **Хранение** | `load.py` | Сохранение в оптимизированных форматах для анализа |
| **Визуализация** | `EDA.ipynb` | Создание интерактивных отчетов и дашбордов |

</div>

### **Результаты работы**

<div align="center">

| Результат | Формат | Назначение |
|-----------|---------|------------|
| **`data/raw/genome_data.csv`** | `CSV` | Сырые данные для анализа и бэкапа |
| **`data/genes.db`** | `SQLite` | База данных для тестирования и демонстрации |
| **`data/processed/genome_data.parquet`** | `Parquet` | Оптимизированные данные для аналитики |

</div>

### **Детали выходных файлов**

- **`data/raw/genome_data.csv`**  
  *Исходные данные после первичной валидации*  
  - Сохраняет структуру оригинального датасета  
  - Используется для отладки и анализа

- **`data/genes.db`**  
  *База данных с примером данных*  
  - Содержит 100 строк для быстрого тестирования  
  - Готова для SQL-запросов и анализа  
  - Показывает как работают запросы к геномным данным

- **`data/processed/genome_data.parquet`**  
  *Оптимизированный формат для анализа*  
  - Высокая скорость чтения  
  - Эффективное сжатие данных  
  - Поддержка сложных типов данных
